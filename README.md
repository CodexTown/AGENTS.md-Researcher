# AGENTS.md — инструкция агента DEEP_RESEARCH

Этот репозиторий хранит локальную инструкцию для агента глубокого анализа (DEEP_RESEARCH). `AGENTS.md` — единственный источник правды: именно оттуда агент читает правила своей работы и на их основе обрабатывает файлы из `_in`, складывая результаты в `_out`.

## Общая картина

- **Агент работает строго локально**: любые сведения извлекаются только из `_in`, внешние источники и интернет запрещены.
- **Документы** в `_in` становятся единственным источником фактов, а в `_out` записываются отчёты, сводки и другие результаты.
- **Вне `_out` ничего не создаётся** (ни логов, ни дополнительных папок), и содержимое `_in` не изменяется.

## Основные этапы работы

1. **Обзор корпуса** — агент перечисляет файлы `_in`, их типы, заголовки и метаданные, чтобы понять контекст.
2. **Целенаправленный поиск** — при запросе агент ищет релевантные фрагменты, фиксирует имя файла, структурный якорь (например, заголовок или пункт списка) и точную цитату.
3. **Глубокий анализ** — на основе найденных фрагментов строится аргументация. Факты и интерпретации отчётливо разделяются, а интерпретации сопровождаются списком источников.
4. **Структурированный вывод** — отчёт оформляется разделами, подпунктами и краткими резюме; каждое значимое утверждение снабжается ссылкой на источник.

## Принцип «никаких додумываний»

- Если данных из `_in` недостаточно, агент прямо говорит об этом и может предложить, какие документы пригодились бы дополнительно.
- Факты приводятся только как прямые цитаты или близкие пересказы (с указанием источника). Интерпретации помечаются как вывод и опираются на конкретные цитаты.

## Ссылки и цитаты

Каждое содержательное утверждение сопровождается ссылкой в формате:

```
[Источник: <имя_файла>, <якорь>, "<короткий фрагмент цитаты>"]
```

Такой формат обеспечивает прозрачность и позволяет проверить любой тезис напрямую по документам.
